# 容量表合成工具 - 功能算法文档

## 1. 概述

本工具用于合并电信运营商的 4G/5G 小区容量报表数据，自动生成综合容量分析表。主要功能包括：

- 自动识别并加载最新的源数据文件
- 按网络制式（4G/5G）分别合成容量表
- 生成 45G 联合总表
- 智能计算流量系数、负荷情况、长尾小区分类等衍生指标
- 通过共站同覆盖关系补充扇区等字段

## 3. 核心数据处理算法

### 3.1 时间字段处理

**判断是否周末：**

```python
day["记录开始时间"] = normalize_datetime(day["记录开始时间"])
day["是否周末"] = day["记录开始时间"].dt.weekday >= 5
```

- `weekday >= 5` 表示周六(5)或周日(6)

**时间戳提取：**

从周表的"记录开始时间"和"记录结束时间"字段提取时间范围，格式化为 `YYYYMMDD_YYYYMMDD` 用于输出文件名。

### 3.3 流量系数计算

```python
# 计算全网日均流量
avg_traffic = result["日均流量"].mean(skipna=True)

# 流量系数 = 小区日均流量 / 全网日均流量
result["流量系数"] = result["日均流量"] / avg_traffic

# 流量排名（升序，流量越小排名越前）
result["流量排名升序"] = result["日均流量"].rank(method="min", ascending=True)
```

**向量化的安全除法：**

```python
def safe_divide(numerator: pd.Series, denominator: pd.Series) -> pd.Series:
    denominator = denominator.replace(0, pd.NA)  # 零值替换为 NA
    return numerator.div(denominator)
```

### 3.4 长尾小区分类算法

**长尾阈值：** 取日均流量的 30% 分位数

```python
tail_threshold = result["日均流量"].quantile(0.3)
```

**分类逻辑（向量化实现）：**

```python
is_na_traffic = result["日均流量"].isna()
is_tail = (result["日均流量"] <= tail_threshold) & ~is_na_traffic
is_zero = result["日均流量"] == 0
is_high_util = result["自忙时利用率"].notna() & (result["自忙时利用率"] > 20)

result["长尾小区"] = pd.NA
result.loc[is_tail & is_zero, "长尾小区"] = "长尾具体原因待确认"      # 低流量且为零
result.loc[is_tail & ~is_zero & is_high_util, "长尾小区"] = "长尾待观察" # 低流量但高负荷
result.loc[is_tail & ~is_zero & ~is_high_util, "长尾小区"] = "长尾需处理" # 低流量且低负荷
```

**分类条件矩阵：**

| 日均流量状态 | 自忙时利用率 > 20% | 分类结果 |
|-------------|-------------------|---------|
| = 0 | 任意 | 长尾具体原因待确认 |
| ≤ 30%分位数 且 ≠ 0 | 是 | 长尾待观察 |
| ≤ 30%分位数 且 ≠ 0 | 否 | 长尾需处理 |

#### 3.6.2 4G 负荷判定（基于小区名称类型）

```python
# 小区名称模式识别
is_rdc_dc = name.str.contains("RDC|DC-|RGS|GS-", regex=True, na=False)  # 数据中心/核心场景
is_rd = name.str.contains("RD-", regex=True, na=False)                  # 农村场景
```

**4G 场景化阈值：**

| 小区类型 | 负荷高阈值 |
|---------|-----------|
| RDC/DC/RGS/GS 开头 | > 90% |
| RD- 开头（农村） | > 70% |

## 4. 扇区映射算法

### 4.1 共站同覆盖映射表加载

```python
def load_cog_coverage_mapping(logger) -> pd.DataFrame:
    # 1. 读取共站同覆盖文件
    df = pd.read_excel(path, dtype_backend="numpy_nullable")

    # 2. 确保必要列存在
    if "CGI" not in df.columns or "共站同覆盖名" not in df.columns:
        return empty_df

    # 3. 去重（同一 CGI 取第一条）
    mapping = df.drop_duplicates(subset=["CGI"], keep="first")

    extra_cols = [col for col in extra_cols if col in df.columns]

    return mapping
```

```python
def apply_sector_mapping(table, mapping, cgi_column, logger):
```

## 5. 表合并算法

### 5.1 5G 容量表构建

**输入源：** `5g_week` + `5g_day` + `5g_mr` + `5g_kpi`

**合并顺序：**
1. 以 `5g_week` 为基础，按 `NCGI` 去重
2. 左连接 `day_group`（天数据聚合）
3. 左连接 `weekday_group`（工作日聚合）
4. 左连接 `weekend_group`（周末聚合）
5. 左连接 `mr_group`（MR 覆盖数据）
6. 左连接 `kpi_group`（KPI 数据）

### 5.2 4G 容量表构建

**输入源：** `4g_week` + `4g_day` + `4g_mr`

**合并顺序：**
1. 以 `4g_week` 为基础，按 `CGI` 去重
2. 左连接 `day_group`（天数据聚合）
3. 左连接 `week_metrics`（周的 PRB 等指标）
4. 左连接 `weekday_group`（工作日聚合）
5. 左连接 `weekend_group`（周末聚合）
6. 左连接 `mr_group`（MR 覆盖数据）

### 5.3 45G 总表合并

```python
def build_45g_table(table_5g, table_4g):

    for col in all_columns:
```

## 8. 性能优化要点

1. **向量化计算**：长尾小区分类、流量是否正常等均使用向量化操作，避免 `apply` 的行迭代开销
2. **分步聚合**：工作日/周末数据通过布尔索引分离后各自聚合，减少数据量
3. **安全除法**：使用 `replace(0, pd.NA)` 避免除零错误
4. **多线程日志**：GUI 中使用独立线程处理数据，主线程更新 UI，通过 Queue 通信
